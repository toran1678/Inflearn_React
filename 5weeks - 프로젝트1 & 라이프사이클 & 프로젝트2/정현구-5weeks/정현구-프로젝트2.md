## 1. Create(생성)

```javascript
const idRef = useRef(3); // 초기 데이터가 0, 1, 2번까지 있으므로 3부터 시작

const onCreate = (content) => {
  const newTodo = {
    id: idRef.current++, // 1. 고유 ID 부여 후 값을 1 증가 (렌더링 방지)
    isDone: false, // 2. 초기 상태는 미완료
    content: content, // 3. 자식에게 받은 내용
    date: new Date().getTime(),
  };

  setTodos([newTodo, ...todos]); // 4. Spread 연산자로 기존 데이터 유지 + 새 데이터 추가
};
```

- useRef 사용이유: ID는 내부적으로 관리되는 값일 뿐, 바뀐다고 다시 그려질 필요가 없으므로 `useState` `useRef`사용하여 불필요한 렌더링 방지
- 불변성 유지: todos.push(newTodo)처럼 원본을 수정하면 리액트가 변화를 감지하지 못해 스프레등 연산자를 통해 새로운 배열을 만들어 전달

## 2. Update : 토글 기능 구현

```javascript
const onUpdate = (targetId) => {
  setTodos(
    todos.map((todo) =>
      todo.id === targetId ? { ...todo, isDone: !todo.isDone } : todo,
    ),
  );
};
```

- 불변성 원칙: 기존 배열을 직접 수정하는 대신, map을 이용해 조건에 맞는 요소만 변경된 새로운 배열을 생성
- 전개 연산자(...todo): 기존 객체의 모든 속성을 유지하면서 특정 속성(isDone)만 덮어씌우는 깔끔한 방식

## 3. Delete: 데이터 삭제

```javascript
const onDelete = (targetId) => {
  // targetId와 일치하지 않는 요소들만 모아서 새로운 배열 생성
  setTodos(todos.filter((todo) => todo.id !== targetId));
};
```

-filter 메서드 활용

## 4. Search: 데이터 검색 및 필터링

```javascript
const getFilteredData = () => {
  if (search === "") {
    return todos; // 검색어가 없으면 전체 반환
  }
  return todos.filter((todo) =>
    todo.content.toLowerCase().includes(search.toLowerCase()),
  );
};

const filteredTodos = getFilteredData();
```

- 원본 todos 배열은 건드리지 않고, 렌더링 직전에만 필터링된 새로운 배열(filteredTodos)을 만들어 사용
- map 함수를 사용할 때는 반드시 고유한 key값으로 todo.id를 전달해야 리액트가 효율적으로 화면을 업데이트
