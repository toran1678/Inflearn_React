# **[JavaScript]** 5주차 라이프사이클 (정현구)

## 라이프사이클이란?

생애 주기를 뜻함. 즉 인간의 탄생부터 죽음까지의 타임라인을 단계별로 나누어 놓는 것

## 리액트 컴포넌트의 라이프사이클

![](https://velog.velcdn.com/images/hyunguu/post/fc43f15a-453b-449d-8351-a375b2658ad4/image.png)

### Mount

- 컴포넌트가 탄생하는 순간
- 화면에 처음 렌더링 되는 순간

ex) A 컴포넌트가 Mount 되었다!  
 ==> A 컴포넌트가 화면에 처음으로 렌더링 되었다.

> 서버에서 데이터를 불러오는 작업

### Update

- 컴포넌트가 다시 렌더링 되는 순간
- 리렌더링 될 때를 의미

ex) A 컴포넌트가 업데이트 되었다!  
 ==> A 컴포넌트가 리렌더링 되었다.

> 어떤 값이 변경되었는 지 콘솔에 출력

### UnMount

- 컴포넌트가 화면에 사라지는 순간
- 렌더링에서 제외 되는 순간을 의미

ex) A 컴포넌트가 언마운트 되었다!  
 ==> A 컴포넌트가 화면에 사라졌다.

> 컴포넌트가 사용하던 메모리 정리

## useEffect

리액트 컴포넌트의 사이드 이펙트를 제어하는 새로운 React Hook

> 리액트 컴폰넌트의 `사이드 이펙트`: 컴포넌트의 동작에 따라 파생되는 여러 효과

![](https://velog.velcdn.com/images/hyunguu/post/cf238dd2-ab33-4850-88dc-af5e666c38b9/image.png)

- 사이드 이펙트를 제어하는 useEffect

기본 문법

```javascript
useEffect(() => {
  // 실행할 코드 (Side Effect)
}, [의존성 배열]);
```

- 첫번째 인자(콜백 함수): 컴포넌트가 렌더링될 때 실행하고 싶은 작업 (API 호출, 콘솔 로그 등)
- 두번째 인자(deps): 이 배열 안에 넣은 값이 변경될 때만 콜백 함수가 실행됨

사용 예시

```javascript
const [count, setCount] = useState(0);
const [input, setInput] = useState("");

useEffect(() => {
  console.log(`count: ${count} / input: ${input}`);
}, [count, input]);
```

![](https://velog.velcdn.com/images/hyunguu/post/ed912ca6-85a4-4941-bd38-e32082a6f178/image.png)

- 이런 식으로 Count와 input 값이 바뀔 때마다 콘솔에 출력

### 왜 이벤트 핸들러에는 바로 콘솔을 찍으면 안될까?

1. 리액트 상태의 비동기성

리액트의 setCount 같은 상태 변화 함수는 비동기로 동작함. 즉 함수를 호출했다고 해서 바로 값이 바뀐느 것이 아님

2. 이벤트 핸들러에서의 한계

```javascript
const onClickButton = (value) => {
  setCount(count + value);
  console.log(count);
};
```

- 이렇게 할 시 콘솔에 바뀌기 전의 이전 값이 찍힘

## useEffect로 라이프사이클 제어하기

1. 마운트: 탄생

```javascript
useEffect(() => {
  console.log("mount");
}, []);
```

- useEffect는 deps에 있는 값이 변경되어야만 실행이 되기 때문에 콜백함수는 다시는 실행되지 않음.
- 마운트 되었을 때 최초로 한번 실행시키고 싶은 코드가 있을 때 사용

2. 업데이트: 변화, 리렌더링

```javascript
useEffect(() => {
  console.log("update");
});
```

- 특정 값이 변화할때마다 로그를 찍음

> 마운트 제외 진짜 업데이트 시점에만 실행하는 법

`useRef`를 사용해 마운트 여부를 추적

```javascript
const isMount = useRef(false);

useEffect(() => {
  if (!isMount.current) {
    isMount.current = true;
    return;
  }
  console.log("update");
});
```

3. 언마운트: 죽음

```javascript
const Even = () => {
  useEffect(() => {
    return () => {
      console.log("unmount");
    };
  }, []);

  return <div>짝수입니다.</div>;
};
```

- UseEffect 내부에서 return 하는 함수를 클린업 함수라고 부름. 이 함수는 컴포넌트가 언마운트 될 때 자동 실행됨.

> 언마운트 제어 중요한 이유?

1. 타이머 제거: setInterval로 만든 타이머가 컴포넌트가 사라진 뒤에도 돌아가면 메모리 낭비가 발생
2. 이벤트 리스너 해제: window.addEventListener로 등록한 이벤트를 지워주지 않으면 성능이 저하
